Cho bộ data nằm ở Data/, dưới đây là bài toán được cung cấp của khách hàng:

Bộ môn: Kinh Tế Lượng
Một số tin nhắn được khách hàng gửi, có thể là gợi ý về bài toán và yêu cầu:

1) "Model"
```
ln(Outputit​)=β0​+β1​ln(Laborit​)+β2​ln(Capitalit​)+β3​Leverageit​+β4​ln(Wageit​)+β5​ln(Sizeit​)+εit
```

2) Factors affecting output of firms in Vietnam’s furniture industry

3) Biến
```
ln(Output_it): log of firm output.

β0: constant term.

β1: coefficient of labor.

ln(Labor_it): log of number of employees.

β2: coefficient of capital.

ln(Capital_it): log of fixed assets.

β3: coefficient of leverage.

Leverage_it: ratio of total debt to total assets.

β4: coefficient of wage.

ln(Wage_it): log of average labor cost per employee.

β5: coefficient of size.

ln(Size_it): log of firm size, measured by total assets.

ε_it: error term.
```

Khách bảo bài toán này data rất nặng nên bạn thấy khó hiểu là chuyện bình thường, tuy vậy thì chỉ cần làm sao để chạy được code/phân tích là được.

Tôi đoán là khi làm bài bạn nên dùng jupyter notebook vì nó khá phổ thông và tiện dụng, tuy vậy thì bạn có thể phân nhỏ ra các mã python và có một file script bash để chạy một phát là xong cả pipeline. Bạn nên chia file rõ ràng để thầy giáo/ giám khảo dễ check và chấm, nói chung là tiện nhất có thể. Sau đó xong xuôi thì bạn hãy làm một file README.md để tổng hợp lại kết quả, các phân tích và ghi chú về phân bố các thư mục.

Trong quá trình làm thì bạn có thể làm mọi cách để xử lý được data, các file excel khá lỗi font nên bạn hãy cố gắng đọc được nội dung, có lẽ quan trọng vẫn là dữ liệu. Nói chung bạn hãy tư duy thật tốt để làm sao cho các file code, các file kết quả, các file hình vẽ biểu đồ... được sắp xếp và trình bày sạch sẽ, hiệu quả, ngăn nắp nhất có thể.

Mặc dù đã cố gắng hết sức để extract các file zip, rar... nhưng có quá nhiều file cần take care. Do đó bạn hãy linh động và chủ động extract chúng, tự làm sạch data, sau đó cứ thế phân tích và chạy data nhé.
